- - - RST ROF MOF ONTRIGGER SAVE
  • SAVE [read/write]: Save data received
    Bit that indicates if the data received is saved(1) in the memory or not(0).
  • ONTRIGGER [read/write]: Save at trigger or continous saving
    This bit indicates that only 1 data for ID is saved for trigger (1) or that is saved all the data after the trigger(0).
  • MOF [read-only]: Memory Overflow
    This bits indicates that a memory overflow is produced and some data can be lost.
  • ROF [read-only]: Memory Overread
    This bits indicates that a memory was red without valid data. So some of the data red could be not valid. This can be caused because readings are to fast or because there is no data in the buffer
  • RST [write-only]: RESET
    This bit produces a reset of error flags and memory.
  • TRIGID [read/write]: ID of trigger generator
    When this Data ID is received Trigger is generated.
  • ID [read/write]: ID of data to save
    ID of the Data to save once trigger is received. To check if data has to be saved only set bits on IDMASK are checked.
  • IDMASK [read/write]: Mask ID of data to save
    Mask of ID, only the bits set at 1 will be checked to identify the data to save. DATAID & IDMASK == ID & IDMASK
