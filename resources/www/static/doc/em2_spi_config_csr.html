- STATE_MACHINE[2:0] ERROR_CODE[2:0] READY
  • READY [read-only]: Ready
    Its value is '0' while SPI bus is busy, '1' when the communication has ended and bus is free.
  • ERROR_CODE [read-only]: Error Code
    Shows if the SPI module state machine ended with an error:
    0 - No error
    1 - Requested TX with length 0
    2 - Requested RX with length 0
    3 - Requested TX and RX with both lengths 0
    4 - No TX or RX requested after address decoding
  • STATE_MACHINE [read-only]: State Machine status
    Shows the current state of the State Machine that manages SPI bus communication:
    0 - IDLE
    1 - WAIT_1
    2 - ADDR_GEN_1
    3 - ADDR_GEN_2
    4 - TX_ON
    5 - RX_ON

- RX RX_DATA_LENGTH[5:0]
  • RX_DATA_LENGTH [read/write]: Reception Data Length
    Length of the data to receive via SPI bus. After SPI communication ends, in the spi_config_ctl_rx_data register the less significant bits indicated by this field will contain received data. Accepatble values from 0 to 32.
  • RX [read/write]: Reception Request
    This bit indicates if a data reception is required.
  • TX_DATA_LENGTH [read/write]: Transmission Data Length
    Length of the data to transmit. From the 32-bit length spi_config_ctl_tx_data register, only the less significant bits indicated by this field will be transmitted. Accepatble values from 0 to 32.
  • TX [read/write]: Transmission Request
    This bit indicates if a data transmission is required.
  • SPI_ADDR [read/write]: SPI Address
    Address of the communication target device. Up to 32 devices can be selected.
  • START [read/write]: Start SPI communication
    This bit starts SPI communication when is set to '1'. All the fields in the spi_config_ctl and spi_config_ctl_tx_data registers must be correctly set and stable before this bit is set to '1', in order to make a correct communication procedure.
