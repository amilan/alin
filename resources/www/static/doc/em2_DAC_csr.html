3.1. DAC Config

WVALUE[7:0]
  • WVALUE [read/write]: Write Value
    Value to write in the DAC. The output depends on the GAIN pin controlled by em# SPI, when GAIN=0 a value 0 means an em# output of 0V and 0xFFFF 10V. when GAIN=1 a value 0 means an out of -10V and 0xFFFF is 10V.
  • WCH [read/write]: DAC CHANNEL to Write
    DAC Channel where 'Value' will be write. 00=channel 1.
  • WR [write-only]: write
    When this bit is '0', the output value is the one written in Value, else the DAC output will be set by HARMONY messages with 'HARMONY ID in'.
  • RST [write-only]: reset
    Reset the Write Overflow flags
  • OW [read-only]: DAC Overwrite
    At least one DAC value is lost. First bit, first DAC channel.

  • 1 [read/write]: DAC1 HARMONY ID in
    Harmony ID that sets the DAC1 output value. Only the 16 bits more significant are used.
  • 2 [read/write]: DAC2 HARMONY ID in
    Harmony ID that sets the DAC2 output value. Only the 16 bits more significant are used.

    Harmony ID that sets the DAC3 output value. Only the 16 bits more significant are used.

    Harmony ID that sets the DAC4 output value. Only the 16 bits more significant are used.
